Getting Started

I began by trying to follow the instructions from these resources:

  • Blogging with R Markdown, by Kevin Wong http://kevinfw.com/post/blogging-with-r-markdown/
  • Building a Blog with Blogdown and GitHub, by Tyler Clavelle https://tclavelle.github.io/blog/blogdown_github/
  • Announcing Blogdown, by Yihui Xie https://blog.rstudio.com/2017/09/11/announcing-blogdown/, along with https://github.com/rstudio/blogdown
  • Blogging with Rmarkdown, knitr, and Jekyll, by Brendan Rocks, https://www.r-bloggers.com/blogging-with-rmarkdown-knitr-and-jekyll/
  • Happy Git and GitHub for the useR, by Jennifer Bryan, https://speakerdeck.com/jennybc/happy-git-and-github-for-the-user

After many false starts and nearly giving up, things finally started falling into place.

In layouts/partials/widgets/posts.html, I changed the capitalization of params

from:

    {{ $params := dict "post" . }}
    {{ partial "post_li" $params }}

to:

    {{ $Params := dict "post" . }}
    {{ partial "post_li" $Params }}

In the file, layouts/partials/widgets/projects.html...

...the first line, below, was trying to delimit $page.Params.tags " ", but failing when there were no tags:

Customization and Personalization

Next, I turned my attention to the About page

How to add a Sub-Section...

This required modification of a few files:

  1. about.md
  2. about.html
  3. en.yaml

about.md

The TOML (Tom's Obvious, Minimal Language) front matter in about.md invites one to 'List your academic interests.'

[interests]
  interests = [
    "Machine Learning",
    "Image Processing",
    "Video Captioning"
  ]

I added a similar section:

[pets]
  pets = [
    "Rex",
    "Fido",
    "Rover"
  ]

In, about.html, noting the interests section:

      {{ with $page.Params.interests }}
      <div class="col-sm-5">
        <h3>{{ i18n "interests" | markdownify }}</h3>
        <ul class="ul-interests">
          {{ range .interests }}
          <li>{{ . | markdownify }}</li>
          {{ end }}
        </ul>
      </div>
      {{ end }}

I added a similar section for pets:

      {{ with $page.Params.pets }}
      <div class="col-sm-5">
        <h3>{{ i18n "pets" | markdownify }}</h3>
        <ul class="ul-pets">
          {{ range .pets }}
          <li>{{ . | markdownify }}</li>
          {{ end }}
        </ul>
      </div>
      {{ end }}

And, in en.yaml, beneath the widget associated with the About section:

# About widget

- id: interests
  translation: Interests

- id: education
  translation: Education

I appended:


- id: pets
  translation: Pets
